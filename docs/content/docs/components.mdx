Interactivity is what makes a Discord bot feel alive. `djs-kit` takes the pain out of managing buttons, modals, and select menus by deeply integrating them into the builder pattern and solving component state with signed custom IDs.

## The State Serialization Problem

In standard discord.js, when you send a button, you give it a `customId` string (e.g. `confirm_ban`). When the user clicks the button, your bot receives `confirm_ban` back.

But what if you need to know *who* they are banning? You'd typically have to stuff that data into the `customId` string manually like `confirm_ban:1234567890`, and then manually split the string later. It gets messy fast.

### The djs-kit Solution: `.addParam()` and Signing

`djs-kit` components allow you to define exactly what parameters a component expects. It serializes the data into the `customId`, signs it, and verifies it before your handler runs.

## Buttons

Generate a button snippet by running `djs-kit add button confirmBan`.

```typescript
// src/components/buttons/confirmBan.ts
import { createButton } from '../../builders/index.js';
import { buildCustomId } from '../../lib/customId.js';

// Usage elsewhere:
// const btn = new ButtonBuilder().setCustomId(
//   buildCustomId('confirm_ban', { targetId: '12345' }, { expiresIn: 300 })
// )

export default createButton('confirm_ban')
  .addParam('targetId')
  .setExecute(async (interaction, args) => {
    // `args` automatically extracts the data from the customId string!
    // args.targetId is a typed string
    
    await interaction.reply({ 
      content: `User <@${args.targetId}> has been banned!`, 
      ephemeral: true 
    });
  });
```

Whenever you render this button in a command, use `buildCustomId()` to pass variables to the button safely. You can also scope the ID to a user or guild.

## Modals

Generate a modal snippet by running `djs-kit add modal feedback`.

Modals are slightly different because they take user input rather than predefined parameters. `djs-kit` allows you to define the modal UI using builder helpers, and then automatically extracts the user's submitted values into an object.

```typescript
// src/components/modals/feedback.ts
import { createModal } from '../../builders/index.js';
import { FieldStyle } from '../../builders/types.js';

export default createModal('feedback_modal')
  .setTitle('Submit Feedback')
  .addTextDisplay('Tell us what happened and attach a screenshot if it helps.')
  .addField('summary', { 
    label: 'Summary', 
    style: FieldStyle.Short, 
    required: true 
  })
  .addStringSelect('topic', {
    label: 'Topic',
    options: [
      { label: 'Bug', value: 'bug' },
      { label: 'Feature', value: 'feature' },
    ],
  })
  .addRadioGroup('priority', {
    label: 'Priority',
    options: [
      { label: 'Normal', value: 'normal', default: true },
      { label: 'Urgent', value: 'urgent' },
    ],
  })
  .addImageUpload('screenshot', {
    label: 'Screenshot',
    required: false,
    maxValues: 1,
  })
  .addField('details', { 
    label: 'Details', 
    style: FieldStyle.Paragraph, 
    required: false 
  })
  .setExecute(async (interaction, fields) => {
    // `fields` is typed based on your modal builder calls.
    // fields.summary is `string`
    // fields.topic is `readonly string[]`
    // fields.priority is `string | null`
    // fields.screenshot is a Discord attachment collection or null
    
    await interaction.reply({ 
      content: `Feedback received! Summary: ${fields.summary}`, 
      ephemeral: true 
    });
  });
```

Modal builders can generate the modal UI, so the field definitions and submitted values stay in one file.

### Components v2 Modal Helpers

Available modal helpers:

- `.addField()` for short and paragraph text inputs
- `.addTextDisplay()` for read-only text
- `.addStringSelect()` for dropdowns
- `.addUserSelect()`, `.addRoleSelect()`, `.addChannelSelect()`, `.addMentionableSelect()`
- `.addFileUpload()` for attachments
- `.addImageUpload()` as a convenience prompt for image attachments
- `.addRadioGroup()` for single-choice radio buttons
- `.addCheckboxGroup()` and `.addCheckbox()` for boolean choices

Discord returns uploaded files as attachments. Always validate file type, size, and content before processing a user upload.

## Select Menus

Generate a select menu snippet by running `djs-kit add select rolePicker`.

```typescript
// src/components/selects/rolePicker.ts
import { createSelect } from '../../builders/index.js';
import { SelectType } from '../../builders/types.js';
import type { Role } from 'discord.js';

export default createSelect('role_picker', { type: SelectType.Role })
  .setExecute(async (interaction, values) => {
    // Since this is a Role select menu, `values` is an array of Role objects!
    const roles = values as Role[];
    
    await interaction.reply({ 
      content: 'Selected roles: ' + roles.map(r => r.name).join(', '), 
      ephemeral: true 
    });
  });
```

Select builders support string, user, role, channel, and mentionable selects. Typed values are resolved before your handler runs wherever Discord provides resolved data.
