The `djs-kit` CLI provides commands for scaffolding entirely new bots and generating code snippets for commands and components inside existing bots.

## `djs-kit create`

Generates a brand new discord.js bot project in the specified directory.

```bash
djs-kit create <project-name> [options]
```

By default, if you run `djs-kit create my-bot`, the CLI will launch an interactive flow to walk you through configuring your bot.

### Options (Non-Interactive Mode)

You can bypass the interactive prompt completely by providing flags. This is useful for automated setups!

| Flag | Description | Default |
| --- | --- | --- |
| `--lang <ts\|js>` | Output language. Required to trigger non-interactive mode. | None |
| `--guild-id <id>` | The Discord guild (server) ID to register slash commands to. | None |
| `--db <none\|file\|sqlite\|postgres\|mysql\|mongo\|redis>` | Database/storage preset. | `none` |
| `--preset <bare\|utility\|moderation\|tickets\|community>` | Starter feature preset. | `utility` |
| `--prefix <string>` | The prefix for text commands (e.g. `!`, `?`, `-`). | `!` |
| `--bare` | Alias for `--preset bare`. Gives you a blank slate. | `false` |
| `--no-install` | Skips running `npm install` after generating the template files. | `false` |

> [!NOTE]
> When using non-interactive mode (`--lang`), the bot token must be provided via the `DISCORD_TOKEN` environment variable.

## `djs-kit add`

Generates ready-to-use boilerplate snippets for your bot's features, deeply typed and correctly placed in your project's directory structure!

You must run these commands from *anywhere* inside a valid `djs-kit` project.

### `add command`

Generates a new command file in `src/commands/`.

```bash
djs-kit add command <name> [--type <slash|prefix>]
```

- **`name`**: The name of the command. Can include paths! (e.g., `admin/ban` generates `src/commands/slash/admin/ban.ts`).
- **`--type`**: Generates a `slash` command or a `prefix` command. (Defaults to `slash`).

### `add button`

Generates a new button component in `src/components/buttons/`.

```bash
djs-kit add button <name>
```

- **`name`**: Converts the provided name into a snake_case `customId` automatically!

### `add modal`

Generates a new modal component in `src/components/modals/`.

```bash
djs-kit add modal <name>
```

### `add select`

Generates a new select menu component in `src/components/selects/`.

```bash
djs-kit add select <name>
```

### `add event`

Generates an event file in `src/events/`.

```bash
djs-kit add event <discord-event-name>
```

### `add autocomplete`

Generates an autocomplete handler for a slash command option.

```bash
djs-kit add autocomplete <command-name> <option-name>
```

### `add context`

Generates a user or message context menu command.

```bash
djs-kit add context <name> --type user
djs-kit add context <name> --type message
```

## Project Tools

Run these inside a generated project.

```bash
djs-kit doctor
djs-kit validate
djs-kit env
djs-kit sync-commands
djs-kit upgrade
```

- `doctor` checks that the generated project shape is intact.
- `validate` checks configuration and required environment.
- `env` prints the environment keys the project expects.
- `sync-commands` registers slash and context commands without starting the bot.
- `upgrade` inspects readiness for newer template features.
