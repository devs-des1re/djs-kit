title: Commands Guide

`djs-kit` replaces raw discord.js API objects with deeply typed, chainable Builders. This allows you to write commands rapidly with perfect TypeScript autocomplete.

## Slash Commands

Generate a slash command template by running `djs-kit add command myCommand`. This creates a file at `src/commands/slash/myCommand.ts`.

Slash commands are exported as a default builder from the file:

```typescript
import { createSlashCommand } from '../../builders/index.js';
import { ParamType } from '../../builders/types.js';

export default createSlashCommand('echo')
  .setDescription('Echoes a message back to you')
  .setCategory('Utility')
  .setExamples(['/echo message:hello'])
  .addParam('message', ParamType.String, { 
    required: true, 
    description: 'The message to echo' 
  })
  .addParam('uppercase', ParamType.Boolean, {
    required: false
  })
  .setExecute(async (interaction, args) => {
    // `args` is fully type-safe! 
    // args.message is `string`
    // args.uppercase is `boolean | undefined`

    let content = args.message;
    if (args.uppercase) content = content.toUpperCase();

    await interaction.reply({ content });
  });
```

### Type-Safe Arguments
Whenever you call `.addParam()`, `djs-kit` injects the parameter name and type into the `args` object inside `.setExecute()`. You never need to call `interaction.options.getString('message')` manually!

## Prefix Commands

Prefix commands operate on classic text messages (e.g. `!say hello`). Generate one by running `djs-kit add command say --type prefix`.

```typescript
import { createPrefixCommand } from '../../builders/index.js';
import { ParamType } from '../../builders/types.js';

export default createPrefixCommand('say')
  .setDescription('Send a message to a channel')
  .setAliases(['s'])
  .addParam('channel', ParamType.TextChannel, { required: true })
  .addParam('message', ParamType.String, { required: true })
  .setExecute(async (message, args) => {
    // args.channel is natively resolved to a GuildTextBasedChannel object!
    // args.message is a string!
    
    await args.channel.send({ content: args.message });
  });
```

### Argument Parsing
For prefix commands, `djs-kit` comes with an internal Argument Parser. It natively handles converting the user's raw text (e.g. `!say #general hello world`) into Discord.js objects. 
- It resolves `#channel` mentions to `Channel` objects.
- It resolves `@user` mentions to `User` or `Member` objects.
- It respects quotation marks for multi-word strings!
- If the final parameter is a string, it consumes the remaining text (e.g. `!suggestion create Better search` maps the last string param to `Better search`).

## Shared Builder Methods

Both `createSlashCommand` and `createPrefixCommand` share powerful configuration methods:

- **`.setCooldown(seconds: number)`**
  Applies an automatic user-level cooldown before they can use the command again.
- **`.setCategory(name: string)`**
  Groups commands for generated help output.
- **`.setExamples(examples: string[])`**
  Adds usage examples for help output and documentation.
- **`.setDefaultMemberPermissions(permissions)`**
  Uses Discord-native slash/context command permissions.

```typescript
import { PermissionFlagsBits } from 'discord.js';

.setDefaultMemberPermissions(PermissionFlagsBits.ManageMessages)
```

## Command Modes

Generated projects can run slash-only, prefix-only, or both from `src/config.ts`.

```typescript
export const config = {
  commandMode: 'slash',
};
```

Slash-only mode avoids the privileged Message Content intent for bots that do not need prefix commands.

## Default Messages

Generated projects keep common fallback responses in `src/config.ts`, so you can customize permission, cooldown, and error copy without editing handlers.

```typescript
export const config = {
  messages: {
    commandPermissionDenied: "You don't have permission to use this command. ({reason})",
    componentPermissionDenied: 'You do not have permission to use this {component}.',
    commandCooldown: 'Please wait {seconds}s before using this command again.',
    commandNotImplemented: 'Command logic not implemented.',
    commandError: 'There was an error while executing this command!',
    componentInvalidState: '{reason}',
    componentError: 'There was an error while executing this component!',
  },
};
```

Available placeholders include `{reason}`, `{seconds}`, and `{component}` depending on the message.

## Autocomplete

```bash
djs-kit add autocomplete ping style
```

Autocomplete handlers live in `src/autocomplete/<command>/<option>.ts` and run when Discord asks for option suggestions.

## Context Menus

```bash
djs-kit add context inspectUser --type user
djs-kit add context quoteMessage --type message
```

Context menu commands live in `src/context` and share the same registration flow as slash commands.
